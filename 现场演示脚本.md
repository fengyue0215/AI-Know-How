## Demo 2：Cursor Composer —— 多文件 Socket 重构（5 min）

### 场景
将现有的阻塞式 TCP 服务改为 epoll 非阻塞模式。

### 准备
先在 Demo 项目中创建一个简单的阻塞式 TCP echo 服务器：

```cpp
// include/network/tcp_server.h
#pragma once
#include <string>
#include <functional>
#include <memory>

namespace demo {

class TcpServer {
public:
    using MessageHandler = std::function<std::string(const std::string&)>;
    
    TcpServer(int port, MessageHandler handler);
    ~TcpServer();
    
    void start();  // 当前是阻塞实现
    void stop();

private:
    class Impl;
    std::unique_ptr<Impl> impl_;
};

} // namespace demo
```

```cpp
// src/network/tcp_server.cpp
#include "network/tcp_server.h"
#include <sys/socket.h>
#include <netinet/in.h>
#include <unistd.h>
#include <cstring>
#include <iostream>

namespace demo {

class TcpServer::Impl {
public:
    int port;
    int server_fd = -1;
    MessageHandler handler;
    bool running = false;
};

TcpServer::TcpServer(int port, MessageHandler handler)
    : impl_(std::make_unique<Impl>()) {
    impl_->port = port;
    impl_->handler = std::move(handler);
}

TcpServer::~TcpServer() {
    stop();
}

void TcpServer::start() {
    impl_->server_fd = socket(AF_INET, SOCK_STREAM, 0);
    // ... 简化的阻塞实现
    std::cout << "Server started on port " << impl_->port << "\n";
}

void TcpServer::stop() {
    if (impl_->server_fd >= 0) {
        close(impl_->server_fd);
        impl_->server_fd = -1;
    }
}

} // namespace demo
```

## Demo 3：AI 辅助调试 —— 分析 coredump（3 min）

```
